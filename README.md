- #### 스티브 잡스의 연설
  프로젝트를 시작하기 앞서 애플 디벨로퍼 아카데미가 이야기하는 인재상, 필요 서류의 의미를 이해하고자 관련 내용 검색을 사전에 진행했습니다. "Connecting the Dots"는 스티브 잡스의 스탠포드 대학 연설에서 이야기 한 내용입니다. [블로그](https://studying-it.tistory.com/19)에서의 내용을 빌려 연설을 요약하자면,   
  
  스티브 잡스는 대학 자퇴 후 서체과목을 수강한 것이 추후에 다양한 서체의 컴퓨터를 만드는데 도움이 되었고, 애플 설립 후 쫓겨났을 때 그 순간은 매우 괴로웠지만 덕분에 NeXT라는 회사를 설립하는 계기가 되었다고 합니다. 핵심은 `과거에 일어났던 사건, 경험들이 현재와는 상관관계가 없어보이지만 결국 돌아보면 지금의 삶을 만들어낸 과정이라는 것입니다.`   
  
  프로젝트를 위해 처음으로 찾아본 스티브 잡스의 연설은 제게 정말 와닿는 내용이었습니다. 돌아보면 정말 이것저것 규칙성 없이 한 것만 같은 저는 평소 아쉬운 생각들을 많이 했었습니다. "왜 이 일을 이 때부터 하지 않았을까?", "기회가 있을 때 잡을 수 있는 용기가 필요했을텐데,,," 등의 생각들 말입니다. 하지만 스티브 잡스의 연설 내용에 따라 제가 지금까지 해왔던 경험들은 의미 없는 일들이 아닌, 지금의 나를 만들어주기 위한 과정이었고 이번 기회를 발판삼아 제 점들을 이어보려 합니다! 스티브 잡스의 말처럼 결국 우리는 이 점들이 미래에 어떻게든 연결될 것이라 믿어야 하기 때문입니다.

- #### 내 인생의 연결점은 어떤 것들이 있었을까?
  지금의 나를 만드는데 어떤 과정이 있었을까를 고민하며 가장 먼저 떠오른 키워드는 "즐겁고 요긴한 경험을 주는 것을 좋아하는 사람" 이었습니다. 어렸을 때부터 나로 인해 누군가를 웃게 만들고, 행복하게 만들고, 도움을 줄 수 있다면 무슨 일이든 했던 제 자신을 표현하기 위한 점들을 나열해보았습니다.   
  
  - 종이게임 만들어 놀기 → 초등학교 쉬는시간, 점심시간 마다 종이에 캐릭터, 아이템, 맵 등을 그려 나만의 게임을 만들고 친구들과 함께 플레이
  - 인터넷 개인 방송 → 어렸을 때부터 좋아했던 게임을 직접 플레이하고 시청자들과 소통하며 나만의 플레이, 컨텐츠 제작
  - 소셜미디어 영상 편집 → 실시간 방송 뿐 아닌 게임 리뷰 등의 영상을 재밌게 편집해 업로드
  - 게임 기획자 → 다양한 게임을 플레이하고 리뷰하다보니 나만의 게임을 만들고 싶어져 게임 기획 공부
  - UI 디자이너 → 친구들과 게임을 만들기 위해 업무 롤 세팅 중, UI디자인에도 관심이 생겨 UI 디자인 공부
  - 게임 종합 플랫폼 설계 → 다양한 유저가 모여 게임 정보를 공유하고 소통할 수 있는 커뮤니티를 만들고 싶어 플랫폼 설계 공부
  - UX 리서처 → 좋은 서비스를 만들기 위해서는 훌륭한 사용자 경험 설계의 필요성을 느껴 UX 공부
  - iOS 개발자 → 기획, 디자인 부분에서의 기초를 다지고 나니 만들어낸 시안들을 실제 프로덕트로 만들어내고 싶어 iOS 개발 공부

- #### Swift에서 점과 점을 이어보기
  우선 인터랙티브한 구현에 있어 가장 중요한 '점과 점 사이를 잇는 선'을 그리는 코드를 작성해보기로 했습니다. Swift에서는 드로잉을 위해 `UIBezierPath`를 사용한다고 합니다. 처음 사용해보는 클래스로 시행착오가 조금 있었지만 첫번째 목표는 이루었습니다.

  ~~~swift
  override func draw(_ rect: CGRect) {
      super.draw(rect)

      // UIBezierPath 객체 생성 및 커스텀
      let path = UIBezierPath()
      UIColor.black.set()
      path.lineWidth = 5

      // 노란점, 초록점 위치 구하기
      let yellowDotX: CGFloat = yellowDot.frame.origin.x + yellowDot.frame.width / 2
      let yellowDotY: CGFloat = yellowDot.frame.origin.y + yellowDot.frame.height / 2
      let greenDotX: CGFloat = greenDot.frame.origin.x + greenDot.frame.width / 2
      let greenDotY: CGFloat = greenDot.frame.origin.y + greenDot.frame.height / 2

      // 시작점 설정
      path.move(to: CGPoint(x: yellowDotX, y: yellowDotY))

      // 끝점 설정
      path.addLine(to: CGPoint(x: greenDotX, y: greenDotY))

      // 선 그리기
      path.stroke()
  }
  ~~~

- #### Pinch(확대, 축소), Pan(드래그) 이벤트 추가하기
  우주 공간을 자유롭게 돌아다니는 느낌을 주고 싶어 이리저리 확대하고 이동하는 이벤트를 구현해보았습니다. 평소에 `UITapGesture` 밖에 사용해보지 않았었는데, 이번 기회로 다양한 제스처를 사용해 볼 수 있었습니다.

  Pinch 제스처
  ~~~swift
  // GestureReconizer 객체 생성
  let pinchGesture = UIPinchGestureRecognizer(target: self, action: #selector(pinchEvent))
  view.addGestureRecognizer(pinchGesture)

  @objc func pinchEvent(gesture: UIPinchGestureRecognizer) {
      // 이벤트 발생 시 Scale값 조정
      let scale = gesture.scale
      view.transform = CGAffineTransform(scaleX: scale, y: scale)
  }
  ~~~

  Pan 제스처
  ~~~swift
  // GestureReconizer 객체 생성
  let panGesture = UIPanGestureRecognizer(target: self, action: #selector(panEvent))
  view.addGestureRecognizer(panGesture)

  @objc func panEvent(gesture: UIPanGestureRecognizer) {
      // 이벤트 발생 시 pan 거리 계산으로 view 위치 조정
      let translation = gesture.translation(in: view.superview)

      // view 위치 조정
      view.center = CGPoint(x: view.center.x + translation.x,
                            y: view.center.y + translation.y)

      // 제스처 종료 시 뷰의 현재 위치 이동 지점으로 초기화
      gesture.setTranslation(CGPoint.zero, in: view.superview)
  }
  ~~~

    어린 시절 쉬는 시간마다 반 친구들 모두가 제 자리에 모이곤 했습니다. 종이 속에서 펼쳐지는 모험을 즐기기 위해서였습니다. 즐거워하는 친구들 모습이 마냥 좋아, 밤을 새워 게임을 만들기도 했습니다. 그때 처음으로 직접 만든 콘텐츠가 사랑받을 때 얼마나 행복할 수 있는지 느끼게되었습니다. 이후 자연스럽게 게임과 관련된 일들을 찾아보다, 네모난 블록을 쌓아 무엇이든 만들어낼 수 있는 게임을 이용해 직접 만든 콘텐츠로 2년간 인터넷 개인 방송을 진행하게 되었습니다. 좋아하는 게임으로 사람들과 소통하며 일을 할 수 있다는 것은 제게 큰 에너지를 주었습니다. 그리고 처음으로 시청자들의 비판적인 피드백을 들으며 콘텐츠 제작의 어려움과 진중함에 대해 알게 되었습니다. 사람들이 사랑하는 콘텐츠는 절대로 쉽게 만들어지지 않는다는 사실로부터, 끊임없는 탐구와 개발을 향한 노력의 순수한 가치에 대해 깨닫게 되었습니다.

  - #### 정답은 있지만 방향성은 있는 디자인
    게임 속에서 콘텐츠를 만들며 인터넷 개인 방송을 하다 보니 그 안에서 모든 상상력을 펼치기 어려워짐을 느꼈습니다. 어린 시절 종이에 나만의 게임을 만들 때처럼 PC 게임을 만들어보고 싶었고, 무작정 책을 구매해 게임 기획에 대해 공부했습니다. 게임 기획을 진행하다 보니 화면 설계가 필요했고 자연스레 UI 디자인도 공부하게 되었습니다. 이때 UI 디자인의 매력에 푹 빠지게 되었습니다. 멋지고 화려한 디자인만이 전부인 줄 알았던 제게, 직관적이고 사용성을 고려한 디자인이라는 것은 완전히 새로운 분야이자 사용자에게 사랑받는 것이라 확신을 주었기 때문입니다. 그렇게 UI 디자인에 대해 공부하고, WEB/APP UI 레퍼런스를 찾아보면 볼수록 더 큰 매력에 빠지게 된 저는 UI 디자인과 더불어 기획, 영상 등의 다양한 콘텐츠에 대해 공부할 수 있는 ‘뉴미디어콘텐츠과’에 진학하게 되었습니다.

  - #### 좋은 아이템은 좋은 기획 속에서
    뉴미디어콘텐츠과에서의 대학교 생활은 다양한 콘텐츠에 시도할 수 있는 기반을 마련해 주었습니다. 기본적인 디자인 스킬부터 영상 편집과 유니티를 이용한 게임 개발까지 배우며 한정되어 있던 콘텐츠에 대한 견문을 넓힐 수 있었습니다. 그중 서비스 기획 강의는 돌아보았을 때 제게 가장 큰 영향을 주었습니다. 자유롭게 플랫폼 서비스에 대한 조별 아이디어를 구체화시켜 기말고사에 발표를 진행한다는 이야기를 첫 강의에 들었을 때, 콘텐츠 제작에 자신이 있었던 저는 빠르게 팀원들과 아이디어를 구체화시켰습니다. 그리고 첫 발표 당시 받았던 충격은 아직도 기억에 남습니다. 좋을 줄만 알았던 아이템은 허점 투성이었기 때문입니다. 충분한 시장 조사와 사용자 조사 없이 기획한 부분들을 교수님이 피드백 주실 때 너무나도 부끄러웠습니다. 아이템에 대한 자만함이 눈을 가려 사용자들이 진정으로 원하는 목표에 도달하지 못할 수 있다는 사실을 가슴 깊이 깨닫게 되었습니다. 이 일을 계기로 기획의 중요성을 배우게 되었고 탄탄히 설계된 기획 속에서 프로젝트를 진행하는 것이 콘텐츠 제작의 최우선 가치가 되었습니다.

  - #### 산출물에 자신감과 설득력 더하기
    UI 디자인을 진행하며 스스로에게 가장 많은 질문을 던졌던 내용은 ‘해당 디자인이 사용자에게 어떻게 평가될 수 있을까?’였습니다. 디자이너의 입장에서는 항상 최우선으로 정보 구조와 사용성을 고려해 디자인을 진행하지만, 과연 스스로의 의사결정으로 사용자 목표에 부합하는 디자인을 만들 수 있을까?에 대한 부분이 궁금했습니다. 그렇게 UX에 대한 관심도가 높아지게 되었고, 교수님의 추천을 통해 인터넷 강의를 구매 후 수강하게 되었습니다. 강의를 들으며 몇 차례의 실습을 진행하다 보니 궁극적인 답을 얻을 수 있었습니다. 핵심은 디자이너 독단의 의사결정은 위험하다는 것입니다. 사용자 목표는 그 주체가 판단하기에도 어렵기 때문에 면밀한 사용자 관찰 기술을 통해 얻은 인사이트를 바탕으로 수립해야 한다는 사실과, 최종 디자인 의사결정은 구성원들과의 합의를 통해 내려야 한다는 사실을 배우게 되었습니다. 이를 통해 사용자를 사용자보다도 더 잘 알아야 한다는 본질적인 답을 얻을 수 있었고 사용자 관찰을 기반으로 한 근거 있는 디자인을 통해 산출물에 대한 자신감과 설득력을 높일 수 있게 되었습니다.

  - #### 디자이너에서 디발자로 거듭나기
    포트폴리오 준비를 통해, 대학 졸업과 동시에 스타트업의 프로덕트 디자이너로 근무하게 되었습니다. 그렇게 서비스 화면 설계, 사용자 리서치 등의 귀중한 경험을 하며 디자이너로서의 역량을 키워나갈 때 쯤, IT 종사자 분들과 함께 일하며 자연스럽게 개발에 대한 관심이 높아지게 되었고 직접 설계한 디자인으로 개발을 하고 싶다는 생각이 많아지게 되었습니다. 그러던 중 iOS 환경의 MVP APP 개발이 필요하게 되었고, 스타트업의 특성상 다양한 업무를 자율적으로 진행할 수 있었기에 APP 개발에 도전하게 되었습니다. 처음엔 문법도 잘 모를뿐더러 간단해보이는 기능도 구현이 복잡해 어려움을 겪었습니다. 하지만 기본 문법은 출퇴근길 지하철과 자택에서 온라인 수강을 하고, 근무 시간에는 기능 구현을 하며 조금씩 완성시켜나갔습니다. 그렇게 노력 끝에 완성한 MVP모델은 VC들에게 서비스 핵심 기능을 보여줄 수 있었고, 이를 통해 개인적으로도 큰 성장을 할 수 있었습니다. 새로운 분야인 개발에 대한 막연한 두려움을 떨쳐내고, 그동안 꿈꿔왔던 나만의 프로덕트를 실제로 만들어낼 수 있다는 자신감과 기대감을 얻게 되었습니다.
